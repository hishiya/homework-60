EXPRESS SERVER — HOMEWORK 60

ОПИС ПРОЄКТУ Даний проєкт є навчальним серверним застосунком,
розробленим з використанням Node.js та Express.js. Метою роботи є
реалізація REST API відповідно до технічного завдання з використанням
MVC-підходу.

  --------------------------------------------------
  ВИКОРИСТАНІ ТЕХНОЛОГІЇ - Node.js - Express.js -
  npm або yarn
  --------------------------------------------------
  СТРУКТУРА ПРОЄКТУ

  HOMEWORK-60/ │ ├── node_modules/ │ ├── src/ │ ├──
  controllers/ │ │ ├── articleController.js │ │ └──
  userController.js │ │ │ ├── routes/ │ │ ├──
  articleRoutes.js │ │ └── userRoutes.js │ │ │ └──
  app.js │ ├── .gitignore ├── package.json ├──
  package-lock.json └── README.txt
  --------------------------------------------------

ЗАПУСК ПРОЄКТУ

1.  Встановити залежності: npm install або yarn install

2.  Запустити сервер: node src/app.js

Сервер запускається на порту 3000.

  --------------------------------------------------
  API МАРШРУТИ
  --------------------------------------------------
  КОРИСТУВАЧІ

  GET /users Отримання списку користувачів.

  POST /users Створення нового користувача.

  GET /users/:userId Отримання користувача за
  ідентифікатором.

  PUT /users/:userId Оновлення користувача за
  ідентифікатором.

  DELETE /users/:userId Видалення користувача за
  ідентифікатором.
  --------------------------------------------------

СТАТТІ

GET /articles Отримання списку статей.

POST /articles Створення нової статті.

GET /articles/:articleId Отримання статті за ідентифікатором.

PUT /articles/:articleId Оновлення статті за ідентифікатором.

DELETE /articles/:articleId Видалення статті за ідентифікатором.

  --------------------------------------------------
  АРХІТЕКТУРА

  Проєкт реалізований з використанням MVC-підходу: -
  Routes відповідають за маршрутизацію - Controllers
  містять логіку обробки запитів - app.js є точкою
  входу та налаштуванням сервера
  --------------------------------------------------

ВИСНОВОК

Проєкт повністю відповідає технічному завданню. Реалізовано всі
необхідні маршрути, сервер коректно обробляє HTTP-запити та повертає
текстові відповіді.
